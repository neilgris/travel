## 1. 项目定位

一个**纯私人**的旅游记录网站，供作者记录每一次旅行：行程、每天的吃喝玩乐购物住宿交通、花费、配图与日记。

- 无账号系统、无多用户、无对外分享（纯私人）。
- **本地单机运行**，不部署云端（未来可升级）。
- 以文字记录为主，**少量配图**。

## 功能总览（当前已实现）

> **状态快照**：描述网站「现在能干嘛」。功能真正上线/移除时更新，不必每次迭代机械改。

- **旅程管理**：创建 / 编辑旅程（标题、日期、行程段 Leg、同行人、多币种及汇率）；旅程列表卡片概要。
- **行程段 Leg**：有序「出发城市→到达城市 + 出行方式」，城市坐标自动地理编码。
- **每日记录**：按天记录日记 + 多条 Entry（吃饭/游玩/购物/住宿/交通，含标题、金额、币种、配图），同天按添加顺序。
- **多币种花费**：每旅程声明多外币，选币种时自动联网获取实时汇率（可手动覆盖），记录时按币种下拉录入，统一换算人民币。
- **单旅程统计**：饼图（按类别）+ 柱状图（按天）+ 按币种原始汇总。
- **设置**：同行人管理、城市管理。
- **图片上传**：Entry 配图存本地 `uploads/`。

## 2. 技术栈

| 项 | 选型 |
|----|------|
| 语言/后端 | Python + Flask |
| 数据库 | SQLite（单文件，零部署） |

## 3. 数据模型

### 3.1 表结构

**Trip 旅程**
| 字段 | 说明 |
|------|------|
| id | 主键 |
| title | 标题，必填 |
| start_date | 开始日期，必填 |
| end_date | 结束日期，必填 |
| notes | 备注/简介，选填 |

> Trip 的「城市」和「出行方式」**不单独存储**，由 Leg 推导（见 3.2）。

**Leg 行程段**（有序）
| 字段 | 说明 |
|------|------|
| id | 主键 |
| trip_id | 外键 → Trip |
| seq | 序号，决定路线顺序 |
| from_city_id | 外键 → City，出发城市 |
| to_city_id | 外键 → City，到达城市 |
| transport_mode | 枚举：飞机/火车/游轮/自驾/…（可扩充） |

> 不含日期字段，仅靠 seq 排序。

**City 城市**（可跨旅程复用）
| 字段 | 说明 |
|------|------|
| id | 主键 |
| name | 名称，如 札幌 |
| latitude | 纬度（自动地理编码获取） |
| longitude | 经度 |
| country | 国家/地区，选填 |

**Person 同行人**（可跨旅程复用）
| 字段 | 说明 |
|------|------|
| id | 主键 |
| name | 姓名 |
| photo | 照片路径 |

**TripPerson**：Trip ↔ Person 多对多关联表。

**TripCurrency 旅程币种**（一个旅程可多条）
| 字段 | 说明 |
|------|------|
| id | 主键 |
| trip_id | 外键 → Trip |
| currency_code | 币种代码，如 JPY、USD |
| rate | 汇率，方向为 **1 人民币 = ? 外币** |

> 人民币为默认本币，不需在此表登记。

**Day 某天**
| 字段 | 说明 |
|------|------|
| id | 主键 |
| trip_id | 外键 → Trip |
| date | 日期 |
| city_id | 外键 → City，所在城市（选自本旅程 Legs 涉及的城市） |
| diary | 当天日记，选填 |

**Entry 记录条目**（属于某天）
| 字段 | 说明 |
|------|------|
| id | 主键 |
| day_id | 外键 → Day |
| category | 枚举：吃饭/游玩/购物/住宿/交通 |
| title | 标题/名称 |
| description | 描述，选填 |
| amount | 金额 |
| currency_code | 币种：人民币或本旅程声明的外币之一 |
| created_at | 创建时间，用于同天排序（按添加顺序） |

**EntryImage 配图**
| 字段 | 说明 |
|------|------|
| id | 主键 |
| entry_id | 外键 → Entry |
| path | 图片路径（一个 Entry 可多张） |

### 3.2 关键派生与约束

- **旅程涉及的城市** = 所有 Leg 的 from/to 城市去重。
- **旅程的出行方式** = 所有 Leg 的 transport_mode 去重。
- 因此**不需要 TripCity 关联表**。
- **Day 与 Leg 不直接关联**，通过 City 间接相连：Day 选城市时，下拉项 = 本旅程 Legs 中出现过的城市集合。
- **两层分离**：Leg = "怎么走"层（路线/交通/地图）；Day = "每天干了啥"层（日记/记录/花费）。第二版地图只用 Leg，日常记录只用 Day。
- **移动日（如游轮在途）简单处理**：用户自选一个城市（出发地或目的地），不强制 Day 与 Leg 对应，不设"在途"特殊城市类型。

### 3.3 花费换算

- 明细按原币种显示。
- 汇总换算：`人民币金额 = 外币金额 ÷ 汇率`（汇率方向为 1 人民币 = ? 外币）。
- 人民币条目无需换算。

## 4. 页面（第一版）

| 页面 | 内容 |
|------|------|
| **旅程列表** | 卡片概要：标题、日期、城市、总花费（第一版仅列表概要） |
| **旅程详情** | 旅程信息 + 同行人/币种 + 按天展开（每天日记 + 各条记录，同天按添加顺序） |
| **旅程统计** | 饼图（按类别）+ 柱状图（按天）+ 按币种原始汇总 |
| **创建/编辑表单** | 新增/编辑：旅程、行程段、天、记录（含上传图片） |
| **设置** | 同行人管理、城市管理 |

### 菜单
- 一级：`旅程` ｜ `设置`
- `旅程` 下含：创建 / 统计
- `设置` 下含：同行人 / 城市

## 5. 统计（单旅程）

- 仅做**单个旅程**的统计，不做跨旅程全局统计。
- **总花费**：所有条目换算为人民币汇总。
- **按类别**（饼图）：吃饭/游玩/购物/住宿/交通。
- **按天**（柱状图）：每天花费。
- **按币种**：各外币原始花费 + 换算后人民币。

## 6. 版本划分

- **第一版**：完整记录流程 + 单旅程统计，旅程页仅列表概要。
- **第二版**：旅程首页 —— 左侧地图/地球展示所有行程路线（用 Leg + City 坐标绘制），右侧旅程列表概要。
- **未来增强**：城市坐标自动地理编码若第一版来不及，可先手填、后升级。

## 7. 明确不做（YAGNI）

- 账号系统 / 多用户 / 登录。
- 对外分享 / 评论 / 公开博客。
- 云端部署（第一版本地运行）。
- 自动汇率获取（汇率手动按旅程设置）。
- 跨旅程全局统计。
- 同天记录的时间字段与拖拽排序（按添加顺序）。
